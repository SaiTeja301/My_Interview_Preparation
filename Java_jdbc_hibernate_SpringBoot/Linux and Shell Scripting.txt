What is an Operating System (OS)?
---------------------------------
--> An Operating System (OS) is software that acts as a mediator between the user and the computer hardware.
--> It allows users to interact with computers and run the programs/apps.
--> Without an OS, we cannot use any computer system.
--> It provides an environment (platform) for users to operate and manage hardware and software resources

File Permission in Linux
==========================
-> read => r
-> write => w
-> execute => x
In Permissions every file will have 3 sections
=> user (u)
=> group (g)
=> otherusers (o)
user group other
read write execute
  r    w     x
rw-r--r--
user -> read and write
group -> read

Change the file permission
===========================
chmod command
$ chmod u+x alien.txt  ==> Gives execute permission for user
$ chmod g+w aws.txt  ==> Gives write permission for group
$ chmod o+x aws.txt  ==> Gives execute permission for others
$ chmod o-x aws.txt  ==> Removes execute permission for other
$ chmod u-x alien.txt  ==> Removes execute permission for user

Link files in files
====================
==> Similar to shortcut files on windows, In Linux we can create link files 
   -> Hard Link
   -> Soft Link
==> Hard Link
===============
$ ln <originalfile> <linkfile>
$ ln telusko.txt teluskoai.txt
$ ls -l
Check inode number of both files
$ ls -li
9182233 -rw-r--r--. 2 ec2-user ec2-user 124 Sep 30 16:03 telusko.txt
9182233 -rw-r--r--. 2 ec2-user ec2-user 124 Sep 30 16:03 teluskoai.txt
telusko(original file) and teluskoai(link) will have same inode number
Note : 1) If we add new content or update or modify the original file then it will reflect in link file also
2) If we delete the original file then it will have no impact/affect on link file
Soft Link
==========
==> ln -s <originalfile> <softlinkfile>
$ ln -s devops.txt devopsai.txt
$ ls -l
-rw-r--r--. 1 ec2-user ec2-user   0 Sep 30 16:23 devops.txt
lrwxrwxrwx. 1 ec2-user ec2-user  10 Sep 30 16:24 devopsai.txt -> devops.txt 
$ ls -li
9182235 -rw-r--r--. 1 ec2-user ec2-user   0 Sep 30 16:23 devops.txt
9182237 lrwxrwxrwx. 1 ec2-user ec2-user  10 Sep 30 16:24 devopsai.txt -> devops.txt
==> Original and softlink file will have diff inode number and by default new soft link file will have maximum file permission(meaning less permission)
Note : 1) If we add new content or update or modify the original file then it will reflect in link file also
2) If we delete the original file then it will have impact/affect on link file and we cannot access soft link file

CROND
======
CROND => is a daemon process (background process), every minute CROND will be checking if CRON JOBs schedule for the execution 
CRON Job Syntax
----------------
one two three four five six
* * * * * <command/scriptfile>
--> First * ==> minutes (0-59)
--> Second * ==> hour (0-23)
--> Third * ==> day of month (1-31)
--> Fourth * ==> Month of year (1-12)
--> Fifth * ==> Day of Week (0-6)(Sun-Mon)
5 * * * * touch telusko.txt
* * * * * touch telusko.txt
one two three four five six
* * * * * <command/script>
--> First * --> minutes(0-59)
--> Second * --> hour (0-23)
--> Third * --> day of month(1-31)
--> Fourth * --> month of year(1-12)
--> Fifth * --> day of week(0-6/ sun - mon)
Run every day @ 5: am --> 0 5 * * * <task/scriptfile>
Run every day @ 5: pm --> 0 17 * * * <task/scriptfile>
Run on 1st day of month --> 0 8 1 * * <task/scriptfile>
Run 5 min of 9 am of 15th day  15 aug 5th day of week  ==> 5 9 15 8 5 echo "hello"
Run on Every Monday @ 10 am --> 0 10 * * 1 <task/script>
Run on 15th of every month @ mid night (00:00) ==> 0 0 15 * * <task/script>
Run every workday (Mon-Friday) @ 6pm ==> 0 18 * * 1-5 <task/script>
Run every 2 hour starting from top of hour ==> * */2 * * * <task/script>

Absolute vs relative path
===========================
Absolute Path: Giving someone the complete, exact street address (including city, state, zip code) so they can find it no matter where they start.
An absolute path (also called a full path) is the complete and unambiguous location of a file or directory on a file system, starting from the very top of the directory tree
Relative Path: Giving someone directions from where they currently are ("Go two blocks straight, then turn left"). These directions only make sense if you know the starting point.
A relative path specifies the location of a file or directory in relation to your current working directory (CWD). Its meaning changes depending on where you currently are in the file system.
